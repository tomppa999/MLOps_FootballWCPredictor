## Key Decisions (current)

### Feature sets
- **Core** (8 features, <2% NaN after dropna): `elo_diff`, `competition_tier`, `is_knockout`, `is_neutral`, `home_team_rolling_goals_for`, `home_team_rolling_goals_against`, `away_team_rolling_goals_for`, `away_team_rolling_goals_against`
- **Full** (28 features): Core + `ROLLING_SHOT_COLUMNS` (6) + `ROLLING_TACTICAL_COLUMNS` (10) + `TACTICAL_CLUSTER_COLUMNS` (4). Substantial NaN (~1,800–2,100 out of 6,663 rows).
- Only XGBoost uses Full. All others use Core.

### Data splits
- Train: date < 2022-11-20 (~3,800 rows after Core dropna)
- Holdout (QA): 2022-11-20 to 2022-12-18 (89 WC 2022 matches)
- Full: all data (6,663 rows, for production refit)

### Metrics
- **Primary**: RPS (Ranked Probability Score) — lower is better
- **Secondary**: RMSE on predicted expected goals
- All models output (λ_h, λ_a); non-distributional models assume Poisson(predicted_mean)

### MLflow setup
- Tracking URI: `file:./mlruns` (hardcoded constant in `mlflow_utils.py`, NOT reading from `.env`)

### Model 3: BayesianPoisson — `src/models/candidates/bayesian_poisson.py`
- Class name: `BayesianPoissonModel` (or similar); `name` property returns `"bayesian_poisson"`
- Bivariate Poisson (Karlis & Ntzoufras, 2003) as a PyMC generative model
- Latent X1, X2, X3 (independent Poisson components); home goals = X1 + X3, away goals = X2 + X3
- Team strength encoded as feature-weighted linear combination in the log-rate parameters
- Manual `StandardScaler` fit on training data before passing features to PyMC
- The scaler must be stored as an instance attribute so `predict()` can apply the same transform
- Optuna tunes `prior_sigma` (prior std on team strength coefficients), `draws`, `tune_steps`
- Search space already defined in `config.py`: `{"prior_sigma": float, "draws": int, "tune_steps": int}`
- Feature set: Core (8 features)
- `fit()` should run MCMC, store posterior samples; `predict()` uses posterior mean of λ parameters
- Keep MCMC chains small (1 chain, small draws) to be testable without a GPU

### Model 4: SARIMAXModel — `src/models/candidates/sarimax.py`
- Class name: `SARIMAXModel`; `name` property returns `"sarimax"`
- Approach: two independent `statsmodels.tsa.statespace.sarimax.SARIMAX` instances (one for home goals, one for away goals)
- Uses `home_team_match_index` / `away_team_match_index` as the time axis (already in Gold schema)
- Exogenous features: the Core feature columns (standardized)
- `fit()` receives the pre-sorted match-level training matrix (X_train, y_train); internally the model must handle the temporal ordering — Gold is already sorted by `date_utc`
- Optuna tunes `p`, `d`, `q` (integer orders); search space already in `config.py`
- Manual `StandardScaler` fit on exogenous features before SARIMAX
- `predict(X)` applies stored scaler and calls SARIMAX `forecast()` or `predict()` for each match
- Feature set: Core (8 features)
- Note: SARIMAX with arbitrary exogenous X doesn't predict arbitrary future rows naturally — use `apply()` or in-sample prediction with replacement exogenous; keep implementation pragmatic

### Model 8: LSTMModel — `src/models/candidates/lstm.py`
- Class name: `LSTMModel`; `name` property returns `"lstm"`
- Sequence model consuming the last N matches as temporal input; 2-unit output (home_goals, away_goals)
- Build sequence tensors inside `fit()` from the flat X_train/y_train arrays (sliding window)
- Keras LSTM layer(s) + Dense(2) output; clip negative predictions to 0
- `StandardScaler` fit on training data; stored as instance attribute
- Optuna tunes: `units`, `num_layers`, `dropout`, `learning_rate`, `batch_size`
- Search space already in `config.py`
- Cap trials at 20–30; use early stopping within each trial
- Feature set: Core (8 features)
- The model should not require GPU — use CPU Keras

### Model 9: CNNModel — `src/models/candidates/cnn.py`
- Class name: `CNNModel`; `name` property returns `"cnn"`
- Same sequence structure as LSTM; use 1D Conv layers instead
- Keras Conv1D + GlobalMaxPooling1D + Dense(2); clip negative predictions to 0
- `StandardScaler` fit on training data; stored as instance attribute
- Optuna tunes: `filters`, `kernel_size`, `dropout`, `learning_rate`
- Search space already in `config.py`
- Feature set: Core (8 features)

## Key decisions

- **Bivariate Poisson GLM uses Karlis & Ntzoufras (2003)**, not Dixon-Coles. Three latent Poisson components with shared λ3 for positive score dependence.
- **NegBin GLM stays independent marginals.** Bivariate NegBin is substantially harder (requires shared gamma mixing or copula) and the gain is small given weak football score correlation.
- **New Bayesian Bivariate Poisson model** will be added as a 10th candidate (`bayesian_bvp`), separate from the existing independent-marginals `bayesian_poisson`. This enables a clean A/B comparison.
- **Safety clip in `compute_mean_nll`** (not in individual models' predict): clip lambdas to 1e-6 to fix Ridge/SARIMAX/XGBoost inf NLL without altering model outputs.
- **Holdout NLL added as QA metric** for reporting alongside RPS. RPS remains the champion/challenger gate. NLL preserves scoreline-level signal that RPS collapses.
- **Optuna trial counts increased** now that Xcode C compiler issue is fixed (PyTensor compiled backend works). Bayesian Poisson search space widened (draws/tune_steps up to 2000, prior_sigma range 0.1–5.0).
- **Narrow results are expected** — shared feature ceiling (8 core features, Elo dominant), low-signal football domain, RPS collapses score distributions. A discussion note in the report will explain this.

## Root cause of inf NLL models
Ridge, SARIMAX, and some XGBoost trials predict λ=0 (clipped from negative). `poisson.logpmf(goals > 0, λ=0) = -inf`. Fix: clip lambdas to 1e-6 in `compute_mean_nll`, not in model predict methods.

## Traps to avoid
- **`conda run` buffers output silently.** Use `conda activate modelops && python -m ...` or `conda run --no-capture-output`.
- **System mlflow vs conda mlflow.** Use `python -m mlflow ui` from conda env.
- **Chrome HSTS on localhost.** Use `http://127.0.0.1:5000`.
- **PyTensor C backend**: The `PYTENSOR_FLAGS=linker=py` workaround in `bayesian_poisson.py` line 19 is already commented out. The C backend works now. Do NOT re-enable the py linker.
- **Bivariate Poisson logp in PyTensor**: The inner k-sum in `_bvp_log_pmf` (poisson_glm.py) uses NumPy/SciPy ops. The Bayesian version needs this rewritten in PyTensor ops (`pt.gammaln`, `pt.logsumexp`). Get broadcasting right — this is the hardest part of the new model.

### Bivariate Poisson reference
> Karlis, D. & Ntzoufras, I. (2003). Analysis of sports data by using bivariate Poisson models.
> Journal of the Royal Statistical Society: Series D (The Statistician), 52(3), 381–393.
>
> Cite in `poisson_glm.py`, `bayesian_poisson.py`, and `docs/modeling_plan.md`.

### Feature sets
- **Core features** (8, nearly complete across all rows): `elo_diff`, `competition_tier`, `is_knockout`, `is_neutral`, `home_team_rolling_goals_for`, `home_team_rolling_goals_against`, `away_team_rolling_goals_for`, `away_team_rolling_goals_against`
- **Full features** (28, Core + rolling shots + tactical): Core + `ROLLING_SHOT_COLUMNS` + `ROLLING_TACTICAL_COLUMNS` + `TACTICAL_CLUSTER_COLUMNS` from `src/gold/schema.py`
- Only XGBoost (Model 7) uses Full features — all others use Core
- Rationale: rolling goals has only ~108–109 NaN out of 6,663 rows (<2%) so it belongs in Core. Rolling shots (~1,837–2,068 NaN) and tactical columns (~1,675–2,068 NaN) have substantial missing data in earlier seasons and stay Full-only. Only XGBoost handles NaN natively.

### Normalization
- Applied at training time inside each model's pipeline, NOT in Gold layer
- Gold stays at natural scale (model-agnostic)
- sklearn-compatible models: StandardScaler wrapped in sklearn Pipeline (single serializable artifact)
- Non-sklearn models (PyMC, SARIMAX, LSTM, CNN): fitted scaler serialized and logged to MLflow alongside model

### Metrics
- **Primary (gating)**: RPS (Ranked Probability Score) — ordinal-aware, proper scoring rule
  - Computed via analytical Poisson→outcome probability conversion (no Monte Carlo needed)
  - `P(H=h, A=a) = poisson.pmf(h, λ_h) * poisson.pmf(a, λ_a)`, truncated at ~10 goals
  - Sum over grid to get P(home win), P(draw), P(away win)
  - For non-distributional models (Ridge, RF, XGBoost): assume Poisson(predicted_mean)
- **Secondary**: RMSE on predicted expected goals
- **Dropped**: Poisson NLL (only applicable to some models, not useful for cross-model comparison)

### Environment pipeline (3 environments)
1. **Experimental**: Train all 9 candidates on pre-WC 2022 data. Optuna tuning with walk-forward expanding-window CV. Log all trials + best runs to MLflow tagged `stage=experimental`. Compute permutation feature importance. Rank by CV RPS. **Top 4 advance to QA.**
2. **QA**: Retrain top 4 with best hyperparameters on full pre-WC 2022 data. Evaluate on WC 2022 holdout. Log runs tagged `stage=qa`. Report RPS + RMSE. **Best 1 advances to Deploy.**
3. **Deploy**: Refit winner on full Gold dataset (all data through today, including WC 2022). Log as `stage=production-refit` with `evaluation_run_id` parameter linking to the QA run. Register and promote in MLflow model registry.

### Hyperparameter tuning
- **Method**: Optuna (TPE sampler) for ALL 9 models
- Walk-forward expanding-window time-series CV on pre-WC 2022 training data
- WC 2022 holdout rows must NEVER be seen during tuning
- Log every trial to MLflow
- SARIMAX: Optuna searches over (p,d,q) integer orders + exogenous feature subsets
- LSTM/CNN: cap trials (20–30), use early stopping within each trial
- Best config logged as MLflow params on final run for reproducibility

### Continuous training strategy
- Pre-tournament: daily trigger at 4:30 UTC. Retrain when Gold grows by ≥10 rows since last cycle.
- Tournament (June 11 – July 19): model frozen, inference-only every 30 min.
- Retraining threshold tracked as MLflow run parameter `gold_row_count`.

## Milestones (implement one at a time, stop for review after each)

### Milestone 1: Foundation

**Goal**: All shared infrastructure — no models yet, but everything a model needs to plug into.

| Module | Scope |
|--------|-------|
| `base.py` | `BaseModel` ABC with `fit(X_train, y_train)`, `predict(X) → (λ_home, λ_away)`, `get_params()`, `name` property. All 9 models implement this interface. |
| `config.py` | **Core features** (8, nearly complete): `elo_diff`, `competition_tier`, `is_knockout`, `is_neutral`, rolling goals for/against (home + away). **Full features** (28): Core + rolling shots (6) + tactical (10) + clusters (4). Model→feature set mapping. Optuna search spaces per model. |
| `data_split.py` | Load Gold parquet. Time-based split: pre-WC 2022 training set, WC 2022 holdout (QA). Walk-forward expanding-window CV fold generator on the training set. |
| `evaluation.py` | `compute_outcome_probs(λ_h, λ_a)` — analytical Poisson grid → P(H), P(D), P(A), truncated at ~10 goals. `compute_rps(probs, actual_outcome)`. `compute_rmse(predicted_goals, actual_goals)`. Permutation importance wrapper. |
| `mlflow_utils.py` | Create/get experiment. Start run with standard tags (`stage`, `model_name`, `environment`). Log params, metrics, artifacts. Model registry helpers for promote/load. |
| `tuning.py` | Optuna study factory: given a `BaseModel` class, search space, train data, and CV folds, run TPE optimization with walk-forward CV. Log every trial to MLflow. Return best params. |
| **Tests** | `test_data_split.py`: split sizes, date boundaries, no leakage across splits. `test_evaluation.py`: RPS on known examples, outcome probs sum to 1, RMSE correctness. |
| **Deps** | Add `optuna` to `pyproject.toml`. |

### Milestone 2: Core Cross-Sectional Models (5 models)

**Goal**: The 5 models that share the same tabular, match-level data structure. All trainable and evaluable through the common `BaseModel` interface.

| # | Model | Key implementation details |
|---|-------|---------------------------|
| 1 | **Poisson GLM** | Bivariate Poisson (Karlis & Ntzoufras, 2003). Joint likelihood with λ1, λ2, λ3 each parameterized as `exp(β^T X)`. Custom MLE via `scipy.optimize`. StandardScaler in pipeline. Feature set: Core. |
| 2 | **NegBin GLM** | Independent `statsmodels.GLM` with NegativeBinomial family + log link. Two separate models (home, away). StandardScaler in pipeline. Feature set: Core. |
| 5 | **Ridge** | `sklearn.Pipeline(StandardScaler, Ridge)`. Two pipelines via `MultiOutputRegressor` or separate fits. Feature set: Core. |
| 6 | **Random Forest** | `sklearn.RandomForestRegressor` with native multi-output. No scaler needed. Feature set: Core. |
| 7 | **XGBoost** | Two `XGBRegressor` instances (one per target). No scaler needed. Feature set: **Full** (only model using tactical features — handles NaN natively). |

Each model:
- Implements `BaseModel` ABC
- Has an Optuna search space defined in `config.py`
- Can be tuned, trained, and evaluated through the common interface
- Has unit tests (fit on tiny synthetic data, predict returns correct shapes, params logged correctly)

Optuna key hyperparameters per model:

| # | Model | Key hyperparameters | Search approach |
|---|-------|--------------------|-----------------| 
| 1 | Poisson GLM | `alpha` (L2 penalty on coefficients) | Short Optuna study |
| 2 | NegBin GLM | Dispersion `alpha` estimated by statsmodels — minimal tuning | Short Optuna study |
| 5 | Ridge | `alpha` ∈ [0.001, 1000] log-scale | Optuna |
| 6 | Random Forest | `n_estimators`, `max_depth`, `min_samples_leaf`, `max_features` | Optuna |
| 7 | XGBoost | `learning_rate`, `max_depth`, `n_estimators`, `subsample`, `colsample_bytree`, `reg_lambda` | Optuna (largest space) |

### Milestone 3: Experimental → QA → Deploy Pipeline

**Goal**: The orchestration that ties models + evaluation + MLflow together into the 3-environment flow.

| Step | What happens |
|------|-------------|
| **Experimental** | For each of the 5 (eventually 9) models: run Optuna tuning with walk-forward CV on pre-WC 2022 data. Log all trials + best run to MLflow tagged `stage=experimental`. Compute permutation feature importance. Rank by CV RPS. **Top 4 advance to QA.** |
| **QA** | Retrain the top 4 with their best hyperparameters on the full pre-WC 2022 training data. Evaluate on WC 2022 holdout. Log runs tagged `stage=qa`. Report RPS + RMSE. **Best 1 advances to Deploy.** |
| **Deploy** | Refit winner on **full Gold** (all data through today, including WC 2022 + subsequent). Log as `stage=production-refit` with `evaluation_run_id` parameter linking to the QA run that justified selection. Register in MLflow model registry. Promote to Production. |
| **Wire trigger** | Fill in `dispatch_training_or_inference()` in `trigger.py` (lines 254–272): read gold row count, compare against `RETRAIN_THRESHOLD` (10 rows), run full pipeline or inference-only with frozen champion. |

Tests: promotion logic (does best model win?), threshold gating (skip retrain when <10 new rows), MLflow registry mock tests.

### Milestone 4: Advanced Models (4 models)

**Goal**: Add the 4 architecturally distinct models to the candidate pool.

| # | Model | Key implementation details |
|---|-------|---------------------------|
| 3 | **Bayesian Poisson** | Bivariate Poisson (Karlis & Ntzoufras, 2003) as PyMC generative model. Latent X1, X2, X3 with explicit priors on team strength coefficients. MCMC sampling. Manual StandardScaler fit on train split before passing to PyMC. Feature set: Core. |

| 9 | **1D CNN** | Same sequence structure as LSTM, convolutional architecture. 2-unit output. StandardScaler serialized alongside. Feature set: Core (or 2020+ if tactical). |

| # | Model | Key hyperparameters | Notes |

## Traps to Avoid
- Do NOT read `data/gold/matches.csv` — use `data/gold/matches.parquet` only
- Do NOT apply normalization in the Gold layer — always at training time
- Do NOT use future information in features (strict time-aware splits)
- Do NOT use random k-fold CV — must be walk-forward expanding-window for time series
- Do NOT let WC 2022 holdout rows leak into tuning/training
- Do NOT reimplement Elo formulas
- Do NOT add Databricks/Spark/Airflow/K8s/dbt/Feast without explicit approval
- The bivariate Poisson GLM (Model 1) is custom MLE via scipy — NOT a statsmodels GLM
- `FEATURE_COLUMNS` in `src/gold/schema.py` includes ALL features — only XGBoost uses the full set; other models use CORE_FEATURE_COLUMNS (4 features) from `src/models/config.py`
- `requests` is used by trigger/ingestion but missing from `pyproject.toml` — note but don't block on it

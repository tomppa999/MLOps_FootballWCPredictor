## Key decisions made

| Decision | Rationale |
|---|---|
| `trigger.py` is pandas-free | numpy segfaults on this machine in subprocess context (known macOS/ARM + numpy 1.x `_mac_os_check` bug). Pandas-free design means trigger tests work in any env. |
| ELO freshness uses temp dir, not live files | Don't overwrite live data just to check; overwrite only if both conditions pass |
| API-Football ingestion IS the freshness check | Avoids double API call — data downloaded in step 2 is reused if pipeline runs |
| `ELO_OVERWRITE` env var, not CLI flag | `download_elo_tsvs.py` has no argparse; env var is a one-line change with no API surface change |
| 04:30 UTC trigger time | Late enough for Americas matches (UTC-5 latest ≈ 02:00 UTC) and ELO update lag |
| GitHub Actions for scheduling now, Cloud Run Jobs later | Zero new infra needed now; migration is drop-in (same Docker image, same Python module) |
| `dvc commit` + `dvc push` for Silver/Gold (not `dvc add`) | Silver/Gold are pipeline stage outputs in `dvc.yaml`; `dvc add` is only for `data/raw` which was already tracked ad-hoc |

## Traps to avoid

1. **Do NOT `dvc add data/silver` or `dvc add data/gold`** — these are pipeline stage outputs; DVC tracks them via `dvc.lock`, not `.dvc` files. Running `dvc add` on them would conflict with the pipeline definition.

2. **numpy segfault on this machine** — numpy 1.x crashes on macOS/ARM in fresh subprocesses. Any test that imports pandas (including silver/gold tests) will segfault when run as a subprocess. The trigger tests are deliberately pandas-free to avoid this. Do not add pandas imports to `trigger.py` or `fixture_status.py`.

3. **`OVERWRITE = False` in `download_elo_tsvs.py`** is now `os.environ.get("ELO_OVERWRITE", "0") == "1"` — this is intentional. Don't revert it to a hardcoded False.

4. **`check_api_football_freshness` returns False if only cancelled/postponed matches** — `summary["finished"] > 0` must be True, not just `settled`. Cancelled-only windows are settled but don't warrant a pipeline run.

5. **Patching in trigger tests** — helpers like `_download_to`, `_sha256`, `_load_slugs` are defined IN `trigger.py` (not imported from elsewhere), so they must be patched at `src.pipeline.trigger._download_to` etc., not at their origin module.

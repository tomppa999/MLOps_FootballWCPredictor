## Traps to avoid
- **`conda run` buffers output silently.** Always use `conda activate modelops && python -m ...` or `conda run --no-capture-output`.
- **System mlflow vs conda mlflow.** Running `mlflow ui` after `conda activate` may still pick up `/Library/Frameworks/Python.framework/Versions/3.12/` system mlflow. Use `python -m mlflow ui` to force the conda env's version.
- **Chrome HSTS on localhost.** Use `http://127.0.0.1:5000` not `http://localhost:5000`.
- **MLflow UI may not show runs** if the system mlflow version differs from the conda mlflow (3.10.0) that wrote the store. Always launch UI from the conda env.
- **API-Football 2-day lookback window** is designed for daily scheduled runs. For manual catch-up after days of inactivity, use `python -m src.ingestion.download_api_football_national_matches --lookback-days 14` first.
- **Ridge and SARIMAX produce inf NLL** across all trials — these models have a predict-path or NLL-computation issue that needs investigation before they can contribute meaningful results.
- **XGBoost, LSTM, CNN** also produce many inf trials — the NLL computation may need clipping or the models may predict non-positive rates in some CV folds.

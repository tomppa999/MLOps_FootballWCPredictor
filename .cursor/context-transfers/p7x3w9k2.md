## Key Decisions (current, not from original plan)

### Feature sets
- **Core** (8 features, <2% NaN after dropna): `elo_diff`, `competition_tier`, `is_knockout`, `is_neutral`, `home_team_rolling_goals_for`, `home_team_rolling_goals_against`, `away_team_rolling_goals_for`, `away_team_rolling_goals_against`
- **Full** (28 features): Core + `ROLLING_SHOT_COLUMNS` (6) + `ROLLING_TACTICAL_COLUMNS` (10) + `TACTICAL_CLUSTER_COLUMNS` (4). Substantial NaN (~1,800–2,100 out of 6,663 rows).
- Only XGBoost uses Full. All others use Core.

### Data splits
- Train: date < 2022-11-20 (~3,800 rows after Core dropna)
- Holdout (QA): 2022-11-20 to 2022-12-18 (89 WC 2022 matches)
- Full: all data (6,663 rows, for production refit)

### Metrics
- **Primary**: RPS (Ranked Probability Score) — lower is better, 0 = perfect, 1 = worst
- **Secondary**: RMSE on predicted expected goals
- All models output (λ_h, λ_a); non-distributional models assume Poisson(predicted_mean)

### MLflow setup
- Tracking URI: `file:./mlruns` (local file store, no server needed)
- Experiment name: `wc_prediction`
- Registry model name: `wc_champion`
- Tags: `stage` ∈ {`experimental`, `qa`, `production-refit`}, `model_name`

## Milestone 3 — What to Build

### Goal
The orchestration pipeline (`src/models/pipeline.py`) that ties models + evaluation + MLflow together into the 3-environment flow, plus wiring the trigger.

### pipeline.py — three phases

**1. Experimental phase**
For each of the 5 models:
- Look up feature set from `config.MODEL_FEATURE_SETS`
- Call `make_splits(df, feature_cols, dropna=...)` to get data
- Generate CV folds via `walk_forward_cv(len(X_train))`
- Call `run_tuning(model_cls, search_space, X_train, y_train, cv_folds, n_trials=...)`
- Refit best params on full training data, compute permutation importance
- Log everything to MLflow tagged `stage=experimental`
- Collect each model's best CV RPS
- **Top 4 by CV RPS advance to QA**

**2. QA phase**
For the top 4:
- Retrain with best hyperparameters on full pre-WC 2022 training data
- Evaluate on WC 2022 holdout → compute holdout RPS + RMSE
- Log runs tagged `stage=qa`
- **Best 1 by holdout RPS advances to Deploy**

**3. Deploy phase**
- Refit winner on full Gold (all data through today including WC 2022)
- Log as `stage=production-refit` with `evaluation_run_id` parameter linking to the QA run
- Serialize model artifact to MLflow
- Register in MLflow model registry via `register_model()`
- Promote to Production via `promote_to_production()`

### Wire trigger.py
Fill in `dispatch_training_or_inference()` at lines 254–272 of `src/pipeline/trigger.py`:
- Read current Gold row count
- Compare against `gold_row_count` param from last production MLflow run
- If delta >= 10 (RETRAIN_THRESHOLD): run full pipeline
- Else: inference-only with frozen champion
- The stub already exists with a TODO comment describing the flow

### Model serialization consideration
`mlflow_utils.register_model()` expects a logged artifact. The pipeline needs to serialize each model. Options:
- sklearn-compatible models (Ridge, RF): `mlflow.sklearn.log_model()`
- XGBoost: `mlflow.xgboost.log_model()`
- Custom models (BivariatePoisson, NegBinGLM): `mlflow.pyfunc.log_model()` with a custom wrapper, or pickle + `mlflow.log_artifact()`
- Decision: simplest approach is pickle + `mlflow.log_artifact()` for all models (uniform interface). The `BaseModel` ABC doesn't inherit from sklearn, so pyfunc wrapping is more work.
